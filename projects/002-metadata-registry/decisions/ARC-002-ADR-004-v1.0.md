## Context

The Metadata Registry Service must align with the BSW (Beter Samenwerken) architecture principles mandated by the Dutch government for zaak/dossierbeheer. BSW represents a paradigm shift from document-centric to information-centric management.

### BSW Core Principles

1. **Informatieobject Centricity**: Everything is an informatieobject (dataobject + metadata)
2. **Dynamic vs Persistent Storage**: Distinguish between "in bewerking" and "gepersisteerd" states
3. **Informatieobject Catalogus**: Metadata + location references (not content storage)
4. **Context-Aware Search**: User context + information origin context
5. **Object-Level Authorization**: Grants at individual information object level
6. **Metadata Inheritance**: Zaak/dossier metadata inherited by contained objects
7. **AI Enrichment with Validation**: Human validation required before trusted use

### Constraints

- **Mandated**: BSW compliance is required for government zaak/dossier systems
- **Existing Investment**: IOU-Modern has existing document-focused systems
- **Collaboration**: Multi-caretaker scenarios require object-level sharing
- **Compliance**: Woo publication workflow requires metadata completeness

### Current State

The existing IOU-Modern system is primarily document-focused. The Metadata Registry must bridge this gap while enabling future BSW adoption.

---

## Decision

**Selected Option**: Full BSW Architecture Alignment

**The Metadata Registry Service will fully implement BSW architecture principles, serving as the foundation for government-wide zaak/dossierbeheer.**

## Alternatives Considered

### Option 1: Full BSW Alignment (SELECTED)

**Description**: Implement all BSW principles from day 1

**Pros**:
- Future-proof for government mandates
- Enables ketensamenwerking (chain collaboration)
- Proper metadata inheritance for zaakbeheer
- Context-aware search for optimal user experience
- Object-level authorization for collaboration

**Cons**:
- Higher implementation complexity
- More entities and relationships
- Additional validation requirements
- Longer time-to-market

**Impact**: Optimal for long-term government collaboration scenarios

---

### Option 2: Partial BSW Alignment

**Description**: Implement core BSW principles (informatieobject, time validity) but defer advanced features

**Pros**:
- Faster initial delivery
- Lower complexity
- Easier to explain to stakeholders

**Cons**:
- Migration pain when adding features later
- May not support multi-caretaker scenarios
- Limited context-aware search
- Risk of technical debt

**Impact**: Faster delivery but requires rework for full BSW compliance

---

### Option 3: BSW-Agnostic Design

**Description**: Focus on GGHH V2 only, ignore BSW-specific requirements

**Pros**:
- Simplest implementation
- Pure focus on metadata registry

**Cons**:
- Not suitable for zaak/dossierbeheer
- Cannot support government collaboration scenarios
- May require complete redesign for BSW

**Impact**: Fails to address primary government use case

### 2. Dynamic vs Persistent Status

**Workflow**:
```
[Creatie] → [Dynamisch] → [Workflow events] → [Gepersistent] → [Gearchiveerd]
                ↑                              ↓
                └───────[Mutable]───────────────┘
```

**Implementation**:
```rust
pub fn update_informatieobject(
    db: &Database,
    id: &str,
    changes: InformatieobjectUpdate,
    user_id: &str,
) -> Result<(), Error> {
    let entity = get_informatieobject(db, id)?;

    match entity.status {
        InformatieobjectStatus::Dynamisch => {
            // Allow updates
            apply_changes(db, id, changes)?;
        }
        InformatieobjectStatus::Gepersistent => {
            // Reject - must create new version
            return Err(Error::ReadOnly);
        }
        InformatieobjectStatus::Gearchiveerd => {
            // Reject - archived
            return Err(Error::Archived);
        }
    }
    Ok(())
}
```

### 4. Context-Aware Search

**Algorithm**:
```rust
pub fn context_aware_search(
    db: &Database,
    query: SearchQuery,
    user_context: &UserContext,
) -> Result<Vec<SearchResult>, Error> {
    let mut results = execute_fulltext_search(db, &query.term)?;

    // Apply context scoring
    for result in &mut results {
        let mut context_score = 0.0;

        // Match user's active zaak
        if let Some(active_zaak) = &user_context.active_zaak {
            if result.zaak_id.as_ref() == Some(active_zaak) {
                context_score += 2.0;
            }
        }

        // Match user's organization
        if result.organisatie_id == user_context.org_id {
            context_score += 1.0;
        }

        // Match user's domain
        if result.domein.as_ref() == user_context.domein.as_ref() {
            context_score += 0.5;
        }

        // Combine with text relevance
        result.final_score = result.text_relevance * (1.0 + context_score);
    }

    // Sort by combined score
    results.sort_by(|a, b| b.final_score.partial_cmp(&a.final_score).unwrap());

    Ok(results)
}
```

### 5. Object-Level Authorization

**Entity**: InformatieobjectRecht

```rust
pub struct InformatieobjectRecht {
    pub _key: String,
    pub informatieobject_id: String,
    pub user_id: String,  // or group_id
    pub recht_type: RechtType,
    pub granted_by: String,
    pub granted_at: DateTime<Utc>,
    pub geldig_vanaf: DateTime<Utc>,
    pub geldig_tot: Option<DateTime<Utc>>,
}

pub enum RechtType {
    Lezen,      // Read
    Bewerken,   // Write
    Goedkeuren, // Approve
    Verwijderen, // Delete
}

pub fn check_object_access(
    db: &Database,
    object_id: &str,
    user_id: &str,
    required Recht: RechtType,
) -> Result<bool, Error> {
    // Check direct grants
    if has_direct_grant(db, object_id, user_id, required Recht)? {
        return Ok(true);
    }

    // Check inherited zaak rights
    if let Some(zaak_id) = get_zaak_for_object(db, object_id)? {
        if has_zaak Recht(db, &zaak_id, user_id, required Recht)? {
            return Ok(true);
        }
    }

    // Check organization rights
    let org_id = get_object_org(db, object_id)?;
    if has_org Recht(db, &org_id, user_id, required Recht)? {
        return Ok(true);
    }

    Ok(false)
}
```

### 6. Metadata Inheritance

**Edge Collection**: inherits_from

```rust
pub struct InheritsFromEdge {
    pub _from: String,  // informatieobject/...
    pub _to: String,    // zaak/...
    pub inheritance_type: InheritanceType,
    pub fields: Vec<String>,  // Which fields to inherit
    pub geldig_vanaf: DateTime<Utc>,
    pub geldig_tot: DateTime<Utc>,
}

pub enum InheritanceType {
    Overerven,   // Inherit only if not set
    Overschrijven, // Always inherit
}

pub fn get_inherited_metadata(
    db: &Database,
    object_id: &str,
) -> Result<HashMap<String, Value>, Error> {
    let mut metadata = HashMap::new();

    // Get object's own metadata
    let obj = get_informatieobject(db, object_id)?;
    metadata.insert("naam".to_string(), json!(obj.naam));
    // ... other fields

    // Traverse inheritance chain
    for inherited in traverse_inheritance(db, object_id)? {
        match inherited.inheritance_type {
            InheritanceType::Overerven => {
                // Only add if not already set
                for field in &inherited.fields {
                    if !metadata.contains_key(field) {
                        metadata.insert(field.clone(), inherited.value[field].clone());
                    }
                }
            }
            InheritanceType::Overschrijven => {
                // Always override
                for field in &inherited.fields {
                    metadata.insert(field.clone(), inherited.value[field].clone());
                }
            }
        }
    }

    Ok(metadata)
}
```

```rust
pub struct AIEnrichment {
    pub _key: String,
    pub informatieobject_id: String,
    pub enrichment_type: EnrichmentType,
    pub input_text: String,
    pub output_text: String,

    // Validation fields
    pub ai_status: AIStatus,
    pub vertrouwensscore: f64,  // 0.0 - 1.0
    pub getoetst_door: Option<String>,
    pub getoetst_op: Option<DateTime<Utc>>,
    pub alternatieve_waarde: Option<String>,

    pub gemodelleerd_door: String,  // AI model
    pub gemodelleerd_op: DateTime<Utc>,
}

pub enum AIStatus {
    Ongecontroleerd,  // AI-generated, not yet reviewed
    Getoetst,        // Human-approved
    Afgewezen,       // Human-rejected
}

pub fn requires_human_validation(
    enrichment: &AIEnrichment,
    context: &UsageContext,
) -> bool {
    match context {
        UsageContext::WooPublicatie => true,  // Always validate for Woo
        UsageContext::InterneZoek => {
            matches!(enrichment.ai_status, AIStatus::Ongecontroleerd)
                && enrichment.vertrouwensscore < 0.8
        }
        UsageContext::InterneAnalyse => false,  // No validation needed
    }
}
```
