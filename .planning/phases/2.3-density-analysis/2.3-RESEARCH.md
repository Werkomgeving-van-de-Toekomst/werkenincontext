# Phase 2.3: Density Analysis - Research

**Researched:** 2026-03-08
**Domain:** MapLibre GL JS Heatmap Visualization + Spatial Density Calculation
**Confidence:** MEDIUM

## Summary

Density visualization in MapLibre uses the `heatmap` layer type, which renders point data as a continuous color gradient representing density. The key implementation challenge is avoiding tile boundary artifacts (PITFALL-03) by using buffered density calculations with overlap.

**Primary recommendation:** Use client-side GeoJSON point aggregation with MapLibre's built-in heatmap layer, calculating density on a 100x100 meter grid with a 50-meter buffer at tile boundaries to prevent seams. Backend density calculation via DuckDB is an optimization for future phases.

### Pattern 1: MapLibre Heatmap Layer (Client-Side)

**What:** MapLibre's `heatmap` layer type with point GeoJSON source

**When to use:** Viewport-based density with < 1000 buildings per viewport

**Example:**

```javascript
// Source: MapLibre GL JS Style Specification (HIGH confidence)
map.addSource('density-points', {
    type: 'geojson',
    data: {
        type: 'FeatureCollection',
        features: [
            {
                type: 'Feature',
                geometry: { type: 'Point', coordinates: [5.5, 52.4] },
                properties: { weight: 1.0 }
            }
        ]
    }
});

map.addLayer({
    id: 'density-heatmap',
    type: 'heatmap',
    source: 'density-points',
    paint: {
        // Color ramp from low (light) to high (dark/intense) density
        'heatmap-color': [
            'interpolate',
            ['linear'],
            ['heatmap-density'],
            0, 'rgba(230, 240, 255, 0)',      // Very low: transparent light blue
            0.2, 'rgba(100, 181, 246, 0.5)',  // Low: light blue
            0.4, 'rgba(33, 150, 243, 0.7)',   // Medium: blue
            0.6, 'rgba(156, 39, 176, 0.8)',   // High: purple
            0.8, 'rgba(74, 20, 140, 0.9)'     // Very high: dark purple
        ],
        'heatmap-radius': 25,
        'heatmap-weight': ['get', 'weight'],
        'heatmap-opacity': 0.7,
        'heatmap-intensity': 1
    }
});
```

### Pattern 2: Density Grid Calculation with Buffer

**What:** Convert building polygons to grid cell centers with overlap at boundaries

**When to use:** Any tile-based density system to avoid edge artifacts

**Algorithm:**

```javascript
// Source: PITFALL-03 research (MEDIUM confidence)
function calculateDensityGrid(buildings, bbox, cellSize = 100, buffer = 50) {
    // Expand bbox by buffer to prevent edge artifacts
    const bufferedBbox = {
        minLon: bbox.minLon - (buffer / 111000),  // Approx 1deg = 111km
        minLat: bbox.minLat - (buffer / 111000),
        maxLon: bbox.maxLon + (buffer / 111000),
        maxLat: bbox.maxLat + (buffer / 111000)
    };

    // Filter buildings in buffered region
    const buildingsInRegion = buildings.filter(b =>
        b.lon >= bufferedBbox.minLon && b.lon <= bufferedBbox.maxLon &&
        b.lat >= bufferedBbox.minLat && b.lat <= bufferedBbox.maxLat
    );

    // Create grid cells
    const grid = {};
    buildingsInRegion.forEach(building => {
        // Calculate which grid cell this building belongs to
        const cellX = Math.floor((building.lon - bbox.minLon) / cellSize);
        const cellY = Math.floor((building.lat - bbox.minLat) / cellSize);
        const key = `${cellX},${cellY}`;

        grid[key] = (grid[key] || 0) + 1;
    });

    // Convert to GeoJSON points (only return cells within original bbox)
    return Object.entries(grid)
        .filter(([key]) => {
            const [cx, cy] = key.split(',').map(Number);
            const centerLon = bbox.minLon + (cx + 0.5) * cellSize;
            const centerLat = bbox.minLat + (cy + 0.5) * cellSize;
            // Only include cells whose center is within original bbox
            return centerLon >= bbox.minLon && centerLon <= bbox.maxLon &&
                   centerLat >= bbox.minLat && centerLat <= bbox.maxLat;
        })
        .map(([key, count]) => {
            const [cx, cy] = key.split(',').map(Number);
            return {
                type: 'Feature',
                geometry: {
                    type: 'Point',
                    coordinates: [
                        bbox.minLon + (cx + 0.5) * cellSize,
                        bbox.minLat + (cy + 0.5) * cellSize
                    ]
                },
                properties: { weight: count }
            };
        });
}
```

### Pattern 3: Building Footprint to Point Conversion

**What:** Extract representative point from building polygon for density calculation

**When to use:** Converting polygon buildings to point grid

**Example:**

```javascript
// Use centroid or first vertex of building footprint
function buildingToPoint(building) {
    if (building.geometry?.type === 'Polygon') {
        const coords = building.geometry.coordinates[0];
        // Simple centroid: average of all vertices
        const lon = coords.reduce((sum, c) => sum + c[0], 0) / coords.length;
        const lat = coords.reduce((sum, c) => sum + c[1], 0) / coords.length;
        return { lon, lat, id: building.id };
    }
    // Fallback for other geometry types
    return null;
}
```

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Heatmap rendering | Custom WebGL shader | MapLibre `heatmap` layer | GPU-optimized, built-in density interpolation |
| Color gradient interpolation | Manual color mixing | `heatmap-color` with `interpolate` | MapLibre handles smooth transitions automatically |
| Point-in-polygon for grid cells | Custom algorithm | Simple grid cell calculation | We don't need precise geometry, just grid approximation |
| Tile boundary detection | Custom edge detection | Buffered bbox queries | Simpler and more reliable |

## Common Pitfalls

### Pitfall 1: Density Tile Artifacts (PITFALL-03)

**What goes wrong:** Visible seams at tile boundaries where density values jump

**Why it happens:** Per-tile calculation without edge consideration means buildings near boundaries only contribute to one tile's density

**How to avoid:** Use buffered bbox queries (expand by 50-100m) and filter results to original bbox after calculation

**Warning signs:** Visible lines or color shifts at tile boundaries, density jumps when panning slightly

### Pitfall 2: Too Many Points degrade Performance

**What goes wrong:** Every building becomes a heatmap point, causing 1000s of points and lag

**Why it happens:** Naive implementation creates one point per building instead of aggregating to grid cells

**How to avoid:** Aggregate buildings to grid cells (100x100m recommended), one point per cell with count as weight

**Warning signs:** Frame rate drops when heatmap is enabled, >5000 points in source

### Pitfall 3: Heatmap Layer Ordering

**What goes wrong:** Heatmap appears below or above other layers incorrectly

**Why it happens:** MapLibre renders layers in array order; heatmap needs to be after buildings but before labels

**How to avoid:** Add heatmap layer with `before` parameter or careful layer ordering

### Pitfall 4: No Feedback During Calculation

**What goes wrong:** User toggles heatmap but nothing appears visible, they think it's broken

**Why it happens:** Density calculation takes time and no loading indicator is shown

**How to avoid:** Show loading spinner when heatmap is enabled and data is being fetched

### Adding Heatmap Layer via JavaScript Interop

```javascript
// Source: Based on existing data_verkenner.rs pattern (HIGH confidence)
function addHeatmapLayer(enabled) {
    const map = window['map_map'];
    if (!map) return;

    if (enabled) {
        // Check if source exists
        if (!map.getSource('density-points')) {
            map.addSource('density-points', {
                type: 'geojson',
                data: { type: 'FeatureCollection', features: [] }
            });
        }

        // Add heatmap layer if not exists
        if (!map.getLayer('density-heatmap')) {
            map.addLayer({
                id: 'density-heatmap',
                type: 'heatmap',
                source: 'density-points',
                paint: {
                    'heatmap-color': [
                        'interpolate',
                        ['linear'],
                        ['heatmap-density'],
                        0, 'rgba(230, 240, 255, 0)',
                        0.2, 'rgba(100, 181, 246, 0.5)',
                        0.4, 'rgba(33, 150, 243, 0.7)',
                        0.6, 'rgba(156, 39, 176, 0.8)',
                        0.8, 'rgba(74, 20, 140, 0.9)'
                    ],
                    'heatmap-radius': 25,
                    'heatmap-weight': ['get', 'weight'],
                    'heatmap-opacity': 0.7
                }
            }, 'building-3d');  // Insert before building layer
        }
        map.setLayoutProperty('density-heatmap', 'visibility', 'visible');
    } else {
        if (map.getLayer('density-heatmap')) {
            map.setLayoutProperty('density-heatmap', 'visibility', 'none');
        }
    }
}
```

### Fetching Density Data

```javascript
// Extend existing fetchBuildings pattern
async function fetchDensityPoints(bbox) {
    const response = await fetch(
        `/api/buildings-3d?bbox-wgs84=${bbox.join(',')}&limit=500`
    );
    const data = await response.json();

    // Calculate grid cells
    const gridCells = calculateDensityGrid(
        data.features.map(buildingToPoint),
        { minLon: bbox[0], minLat: bbox[1], maxLon: bbox[2], maxLat: bbox[3] },
        100,  // 100m cells
        50    // 50m buffer
    );

    // Update heatmap source
    const source = map.getSource('density-points');
    if (source) {
        source.setData({
            type: 'FeatureCollection',
            features: gridCells
        });
    }
}
```
