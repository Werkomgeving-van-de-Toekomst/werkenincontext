## Key Features

### Heatmap Layer Configuration

```javascript
// MapLibre heatmap layer with color gradient
paint: {
    'heatmap-color': [
        'interpolate',
        ['linear'],
        ['heatmap-density'],
        0, 'rgba(230, 240, 255, 0)',      // Very low: transparent
        0.2, 'rgba(100, 181, 246, 0.5)',  // Low: light blue
        0.4, 'rgba(33, 150, 243, 0.7)',   // Medium: blue
        0.6, 'rgba(156, 39, 176, 0.8)',   // High: purple
        0.8, 'rgba(74, 20, 140, 0.9)'     // Very high: dark purple
    ],
    'heatmap-radius': 25,
    'heatmap-weight': ['get', 'weight'],
    'heatmap-opacity': 0.7
}
```

### Grid Aggregation Algorithm

1. Fetch buildings with buffered bbox (50m expansion)
2. Calculate centroid for each building polygon
3. Aggregate to 100x100m grid cells using cell key calculation
4. Filter to cells whose center is within original (non-buffered) bbox
5. Update MapLibre GeoJSON source with aggregated points

### Viewport Event Handling

- Debounced density update function (300ms delay)
- Event listeners: `map.on('moveend')` and `map.on('zoomend')`
- State tracking: lastFetchedBbox, fetchTimeout, abortController
- Only updates when heatmap is enabled (checked via localStorage)
- Follows existing data_verkenner.rs pattern exactly

## Decisions Made

1. **Client-side aggregation only**: No backend density endpoint - uses existing /api/buildings-3d endpoint with client-side grid calculation
2. **Copy trait on DensityHeatmap**: Required for use_signal compatibility (Value must be Copy or Clone for read() dereference)
3. **Renamed state import**: Used `use crate::state::DensityHeatmap as DensityHeatmapState` to avoid naming conflict with component
4. **Component positioning**: Top-right (10px from edge) opposite ViewToggle (top-left) for balanced UI layout
