# Key interface contracts

MapLibre Heatmap Layer API (from RESEARCH.md):
- map.addLayer({ id: 'density-heatmap', type: 'heatmap', source: 'density-points', paint: { 'heatmap-color': [interpolate expression], 'heatmap-radius': 25, 'heatmap-weight': ['get', 'weight'], 'heatmap-opacity': 0.7 }})

Buffered bbox calculation (to prevent tile seams):
- Expand bbox by 50 meters (~0.00045 degrees at Dutch latitudes)
- Query buildings in buffered region
- Aggregate to 100x100m grid cells
- Return only cells whose center is within original bbox

# Map event listener pattern (from data_verkenner.rs)

The existing code uses debounced fetch with event listeners:

```javascript
// Debounced fetch function
function debouncedFetch() {
    clearTimeout(fetchTimeout);
    fetchTimeout = setTimeout(() => {
        const bounds = map.getBounds();
        const bbox = [bounds.getWest(), bounds.getSouth(),
                      bounds.getEast(), bounds.getNorth()];
        if (shouldFetch(bbox)) {
            fetchBuildings(bbox);
        }
    }, 300);
}

// Event listeners registered in map.on('load')
map.on('moveend', debouncedFetch);
map.on('zoomend', debouncedFetch);
```

The density heatmap must follow this same pattern for viewport updates.
