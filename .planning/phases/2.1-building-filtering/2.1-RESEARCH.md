# Phase 2.1: Building Filtering - Research

**Researched:** 2026-03-08
**Domain:** 3D Building Data Filtering with MapLibre GL JS + Dioxus WASM
**Confidence:** HIGH

## Summary

Phase 2.1 requires implementing client-side filtering for 3D building data using MapLibre GL JS filter expressions. The existing codebase already has MapLibre v4.7.0 integration via `eval()` interop pattern in `data_verkenner.rs` and `map_3d.rs`. The filtering implementation must use `map.setFilter()` to avoid PITFALL-01 (Filter re-render cascades) and integrate with the existing AbortController pattern for viewport-based loading (PITFALL-02).

**Primary recommendation:** Use MapLibre's native `setFilter()` with JSON filter expressions, updated via generated JavaScript from Dioxus signals. No new dependencies required.

### Pattern 1: MapLibre Filter Expressions

**What:** MapLibre's declarative filtering using JSON array expressions

**When to use:** Client-side filtering of viewport-loaded features

**Example:**
```javascript
// Single property range filter
map.setFilter('building-3d', [
    'all',
    ['>=', ['get', 'construction_year'], 1950],
    ['<=', ['get', 'construction_year'], 2000]
]);

// Multiple property filters (combined)
map.setFilter('building-3d', [
    'all',
    ['>=', ['get', 'construction_year'], minYear],
    ['<=', ['get', 'construction_year'], maxYear],
    ['>=', ['get', 'height'], minHeight],
    ['<=', ['get', 'height'], maxHeight],
    ['>=', ['get', 'floors'], minFloors],
    ['<=', ['get', 'floors'], maxFloors]
]);

// Clear filter (show all)
map.setFilter('building-3d', null);
```

**Source:** MapLibre Style Specification - Filter expressions

### Pattern 2: Dioxus Reactive State with Signals

**What:** Dioxus `use_signal()` for reactive filter state

**When to use:** UI components that need to update map on user input

**Example:**
```rust
#[component]
fn FilterPanel3D() -> Element {
    // Filter state - use signals for reactivity
    let mut min_year = use_signal(|| 1900);
    let mut max_year = use_signal(|| 2024);
    let mut min_height = use_signal(|| 0.0);
    let mut max_height = use_signal(|| 100.0);
    let mut min_floors = use_signal(|| 1);
    let mut max_floors = use_signal(|| 10);

    // Apply filter when any value changes
    use_effect(move || {
        let filter_expr = build_filter_expression(
            *min_year.read(), *max_year.read(),
            *min_height.read(), *max_height.read(),
            *min_floors.read(), *max_floors.read()
        );
        let script = build_set_filter_script("building-3d", &filter_expr);
        document::eval(&script);
    });

    rsx! {
        div { class: "filter-panel",
            // Year range sliders
            label { "Bouwjaar: {min_year} - {max_year}" }
            input {
                type: "range",
                min: 1800,
                max: 2024,
                value: "{min_year}",
                oninput: move |e| {
                    if let Ok(v) = e.value().parse::<u32>() {
                        min_year.set(v);
                    }
                }
            }
            // ... more sliders
        }
    }
}
```

### Pattern 3: eval() Interop for Filter Updates

**What:** Generate JavaScript strings in Rust and execute via `document::eval()`

**When to use:** Calling MapLibre APIs from Dioxus components

**Example (following existing pattern):**
```rust
/// Builds JavaScript to update filter on a layer
fn build_set_filter_script(layer_id: &str, filter_expr: &str) -> String {
    format!(r#"
        (function() {{
            const map = window['map_map'];
            if (!map) {{
                console.error('Map not found');
                return;
            }}
            const filter = {filter_expr};
            map.setFilter('{layer_id}', filter);
            console.log('Filter updated:', filter);
        }})();
    "#)
}

/// Builds filter expression JSON from state
fn build_filter_expression(
    min_year: u32, max_year: u32,
    min_height: f64, max_height: f64,
    min_floors: u32, max_floors: u32
) -> String {
    // Build the filter expression JSON
    format!(r#"["all",
        [">=", ["get", "construction_year"], {}],
        ["<=", ["get", "construction_year"], {}],
        [">=", ["get", "height"], {}],
        ["<=", ["get", "height"], {}],
        [">=", ["get", "floors"], {}],
        ["<=", ["get", "floors"], {}]
    ]"#, min_year, max_year, min_height, max_height, min_floors, max_floors)
}
```

### Anti-Patterns to Avoid

- **[Anti-pattern]: Recreating layer on filter change**
  - Why: Causes full re-render, 1-3 second freezes
  - Do instead: Use `map.setFilter('layer-id', expression)`

- **[Anti-pattern]: Hardcoded filter values**
  - Why: Not maintainable, can't validate
  - Do instead: Serialize from Rust state, generate dynamically

- **[Anti-pattern]: Separate layers for 2D/3D filtering**
  - Why: State desync, complexity
  - Do instead: Single layer, toggle properties

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Filter expression parsing | Custom JSON parser | MapLibre's native expression syntax | Built-in, GPU-accelerated |
| Debouncing for sliders | Custom timeout logic | Dioxus `use_resource` or simple timeout | Already implemented in viewport loading |
| Range slider UI | Custom dual-handle slider | HTML5 `<input type="range">` with two inputs | Simpler, accessible |

## Common Pitfalls

### Pitfall 1: Filter Re-render Cascades (PITFALL-01)

**What goes wrong:**
Every filter change triggers full re-render of all 3D buildings, causing frame drops and UI freezing (1-3 second delays).

**Why it happens:**
Developers implement filters by clearing and re-adding layers (`map.removeLayer()` + `map.addLayer()`) instead of using `setFilter()`.

**How to avoid:**
```javascript
// BAD - causes full re-render
map.removeLayer('building-3d');
map.addLayer({ id: 'building-3d', filter: [...] });

// GOOD - GPU-accelerated filter update
map.setFilter('building-3d', ['all', ...]);
```

**Warning signs:**
- Frame profiler shows >100ms frames during filter changes
- Visual "flash" or blank map during filter updates

### Pitfall 2: Viewport Race Conditions (PITFALL-02)

**What goes wrong:**
Rapid map panning causes multiple simultaneous API requests. Old data arrives after new data, displaying stale buildings.

**How to avoid:**
Already implemented in `data_verkenner.rs` with AbortController pattern. Reuse this pattern for filter-related API calls if needed.

### Pitfall 3: Filter Expression Complexity

**What goes wrong:**
Combining many filters causes frame rate drops from 60fps to 15fps with 1000+ buildings.

**Prevention:**
- For <1000 buildings: client-side filters are fine
- For >1000 buildings: consider server-side filtering via backend API
- Use `["all", ...]` to combine conditions efficiently

## Code Examples

### Filter Expression Syntax

```javascript
// Source: MapLibre Style Specification

// Get property value
["get", "property_name"]

// Comparison operators
[">=", ["get", "height"], 10]    // height >= 10
["<=", ["get", "height"], 50]    // height <= 50
[">", ["get", "floors"], 3]      // floors > 3
["<", ["get", "floors"], 10]     // floors < 10
["==", ["get", "year"], 2000]    // year == 2000
["!=", ["get", "type"], "residential"]

// Logical operators
["all", condition1, condition2, ...]  // AND
["any", condition1, condition2, ...]  // OR
["!", condition]                      // NOT

// Type assertion (optional safety)
["number", ["get", "height"]]  // Ensures numeric comparison
```

### Building Count Query

```javascript
// Get visible (filtered) building count
function getVisibleBuildingCount() {
    const map = window['map_map'];
    if (!map) return 0;

    const features = map.queryRenderedFeatures({
        layers: ['building-3d']
    });

    return features.length;
}

// Call from Rust after filter update
const count = getVisibleBuildingCount();
// Send back to Rust via window.sendToRust() or update DOM
```

### Complete Filter Update Script

```javascript
// Generated by Rust build_set_filter_script()
(function() {
    const map = window['map_map'];
    if (!map) {
        console.error('Map not found for filter update');
        return { success: false, error: 'Map not found' };
    }

    try {
        const filter = ["all",
            [">=", ["get", "construction_year"], 1950],
            ["<=", ["get", "construction_year"], 2020],
            [">=", ["get", "height"], 5.0],
            ["<=", ["get", "height"], 50.0],
            [">=", ["get", "floors"], 1],
            ["<=", ["get", "floors"], 10]
        ];

        map.setFilter('building-3d', filter);

        // Update visible count
        const count = map.queryRenderedFeatures({
            layers: ['building-3d']
        }).length;

        // Update DOM element or send to Rust
        const countEl = document.getElementById('building-count');
        if (countEl) {
            countEl.textContent = count.toString();
        }

        return { success: true, count: count };
    } catch (e) {
        console.error('Filter update error:', e);
        return { success: false, error: e.toString() };
    }
})();
```
