# Section 02: Template System

## Overview

This section implements the Markdown template rendering system for document generation. The template system enables flexible document creation through variable substitution, conditional sections, and format conversion capabilities.

**Dependencies:**
- section-01-foundation (must be completed first)

**Blocks:**
- section-04-content-agent (requires this section to be complete)

## Key Concepts

### Template Engine Choice: Tera

We use **Tera** as the template engine for the following reasons:

1. **Rust-native**: No external dependencies or CLI invocations
2. **Jinja2-like syntax**: Familiar to developers, easy to author
3. **Type-safe**: Compile-time template validation possible
4. **Performance**: Efficient rendering with minimal overhead
5. **Extensibility**: Custom filters and functions for domain-specific needs

### Template Variable Sources

Template variables can come from multiple sources, with the following priority order:

1. **UserInput** - Direct user input (highest priority)
2. **KnowledgeGraph** - Values retrieved from the knowledge graph
3. **AgentGenerated** - Generated by AI agents during processing
4. **Default** - Fallback default values defined in the template

### Storage Strategy

Templates are stored in DuckDB for dynamic management and versioning. Initial templates are seeded from the filesystem during migrations.

use tera::{Tera, Context, Result as TeraResult};
use std::sync::Arc;
use std::collections::HashMap;
use iou_core::document::{Template, TemplateVariable, VariableSource};

/// Template engine wrapper for document generation
pub struct TemplateEngine {
    tera: Arc<Tera>,
    // Will add DB connection in next task
}

impl TemplateEngine {
    /// Create a new template engine instance
    pub fn new() -> Result<Self, TemplateError> {
        let mut tera = Tera::default();
        
        // Add custom filters for Dutch government documents
        tera.register_filter("dutch_date", dutch_date_filter);
        tera.register_filter("format_iban", format_iban_filter);
        
        Ok(Self {
            tera: Arc::new(tera),
        })
    }
    
    /// Register a template from a string
    pub fn register_template(&mut self, name: &str, content: &str) -> Result<(), TemplateError> {
        self.tera.add_raw_template(name, content)?;
        Ok(())
    }
    
    /// Render a template with the given context
    pub fn render(
        &self,
        template_name: &str,
        variables: &HashMap<String, TemplateVariable>,
    ) -> Result<RenderedDocument, TemplateError> {
        let mut context = Context::new();
        let mut variables_used = Vec::new();
        
        // Add all variables to context
        for (name, var) in variables {
            context.insert(name, &var.value);
            variables_used.push(name.clone());
        }
        
        // Render the template
        let content = self.tera.render(template_name, &context)?;
        
        // Check for missing variables (Tera reports these as errors)
        let missing_variables = self.detect_missing_variables(&content);
        
        Ok(RenderedDocument {
            content,
            variables_used,
            missing_variables,
        })
    }
    
    /// Detect variables that weren't filled (placeholder patterns remaining)
    fn detect_missing_variables(&self, content: &str) -> Vec<String> {
        // Regex to find Tera variables that weren't replaced
        let re = regex::Regex::new(r"\{\{\s*(\w+)\s*\}\}").unwrap();
        re.captures_iter(content)
            .map(|cap| cap[1].to_string())
            .collect()
    }
}

/// Custom filter for Dutch date formatting
fn dutch_date_filter(value: &tera::Value, _args: &HashMap<String, tera::Value>) -> TeraResult<tera::Value> {
    // Implementation for Dutch date format (e.g., "1 maart 2025")
    Ok(tera::Value::String(value.to_string()))
}

/// Custom filter for IBAN formatting
fn format_iban_filter(value: &tera::Value, _args: &HashMap<String, tera::Value>) -> TeraResult<tera::Value> {
    // Implementation for Dutch IBAN formatting (e.g., "NL91 ABNA 0417 1643 00")
    Ok(tera::Value::String(value.to_string()))
}

#[derive(Debug, thiserror::Error)]
pub enum TemplateError {
    #[error("Tera error: {0}")]
    Tera(#[from] tera::Error),
    
    #[error("Variable not found: {0}")]
    VariableNotFound(String),
    
    #[error("Template not found: {0}")]
    TemplateNotFound(String),
}

        agent_data: &HashMap<String, String>,
    ) -> Result<HashMap<String, TemplateVariable>, TemplateError> {
        let mut result = HashMap::new();
        
        for var_name in &template.required_variables {
            let (value, source) = self.resolve_variable(
                var_name,
                user_input,
                kg_data,
                agent_data,
                &template.optional_sections,
            )?;
            
            result.insert(var_name.clone(), TemplateVariable {
                name: var_name.clone(),
                value,
                source,
            });
        }
        
        Ok(result)
    }

}

/// Use pandoc for conversion (external process)
fn convert_with_pandoc(markdown: &str, format: &str) -> Result<Vec<u8>, ConversionError> {
    let mut child = Command::new("pandoc")
        .arg("-f")
        .arg("markdown")
        .arg("-t")
        .arg(format)
        .arg("--standalone")
        .stdin(std::process::Stdio::piped())
        .stdout(std::process::Stdio::piped())
        .spawn()?;
    
    // Write markdown to stdin
    use std::io::Write;
    if let Some(stdin) = child.stdin.as_mut() {
        stdin.write_all(markdown.as_bytes())?;
    }
    
    let output = child.wait_with_output()?;
    
    if output.status.success() {
        Ok(output.stdout)
    } else {
        Err(ConversionError::PandocFailed(
            String::from_utf8_lossy(&output.stderr).to_string()
        ))
    }
}

/// Rust-based ODF generation (fallback)
